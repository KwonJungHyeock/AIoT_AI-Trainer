# Eduino AI Lab — 제품 기획안 (소프트웨어)

> 체험으로 빠르게 이해하는 웹 기반 AI 학습 콘텐츠를, **주제별 모듈 = 개별 판매 상품**으로 양산한다.

---

## 1. 배경 · 목표
- 현재 1차 산출물 = **비전·영상처리 AI 모듈**(이미지/동작/음성 분류 + 영상효과/색상추적/얼굴/뼈대/객체탐지). 잘 묶인 하나의 완성 제품.
- 같은 골격(사이드바·정보카드·StepNav·이론·테스트·다크테마)을 **공통 코어(템플릿)** 로 재사용해, **주제별 모듈을 빠르게 찍어내** 개별 상품화한다.
- 설치 없이 **브라우저 완결(On-Device)** · **Vercel 개별 배포** → 링크만으로 판매·배포.

## 2. 핵심 원칙
1. **쉬운 이해** — K-12 눈높이 설명(학습 정보 / 알고리즘 정보 카드).
2. **체험 기반 빠른 이해** — 카메라·소리·데이터로 즉시 반응, 보고 만지며 학습.
3. **교실 상호작용** — 몸 참여·게임·실시간 피드백(게이지/도넛/단계), 교사-학생 활동 설계.

## 3. 제품 전략 (모듈형 상품화)
- **공통 코어 1개 + 모듈 N개** 구조. 모듈 = 독립 웹앱 = 개별 SKU.
- 모듈 추가 = **config(기능 메타) + 기능 로직만** 작성(코어 재사용).
- 브랜딩: `Eduino AI Lab — 비전 / 제스처 / 데이터 / 언어 / 개념 / 생활`.
- 배포: 모듈별 Vercel 프로젝트(개별 URL/접근).

## 4. 레벨 정의 (전 모듈 공통)
| 레벨 | 방식 | 학습자가 하는 것 |
|---|---|---|
| **초급(체험)** | 사전학습 모델 그대로 + 시각화 | 관찰·원리 이해 |
| **중급(제작)** | 내 데이터 전이학습 + 파라미터 | "내 AI 만들기" |
| **심화(응용·개념)** | 평가·일반화·비지도·연동 | 깊은 개념 + 실생활 응용 |

## 5. 모듈 라인업
| # | 모듈 | 대상(주) | 핵심 기술 | 체험·상호작용 |
|---|---|---|---|---|
| 0 | 비전·영상처리 *(완성)* | 초등~중등 | MobileNet/MoveNet/COCO-SSD/BlazeFace/Canvas | ●●● |
| 1 | 텍스트·언어 | 중등~대학 | Universal Sentence Encoder | ●○○ (보완 필요) |
| 2 | 데이터·예측(AIoT) | 중고~대학 | tf 회귀/MLP, (센서 연동) | ●●○ |
| 3 | 제스처·동작 | 초등~중등 | Hand Pose + 전이학습 | ●●● |
| 4 | ML 개념 실험실 | 고교~대학 | K-means, PCA, 평가지표 | ●●○ |
| 5 | 생활 인식 | 초등~중등 | QR/BarcodeDetector, OCR(Tesseract.js) | ●●● |

## 6. 모듈별 초급/중급/심화
**0 비전(완성)** — 초:객체/얼굴/뼈대 체험 · 중:이미지/동작 분류 학습 · 심:색추적·필터 파라미터

**1 텍스트·언어** — 초:"비슷한 말 찾기"·감정 분류 · 중:내 데이터 주제 분류 · 심:유사문장 검색·임베딩 개념
> 보완: "**말하면 분류**"(음성→텍스트), 단어 맞히기 게임으로 체험·상호작용 강화.

**2 데이터·예측(AIoT)** — 초:산점도·1변수 관계 관찰 · 중:선형회귀·표 분류 · 심:MLP·과적합/일반화·(센서 실시간)
> 보완: "**학급 데이터 실시간 수집**"(학생 입력→즉석 산점도)로 상호작용↑.

**3 제스처·동작** — 초:손추적·가위바위보(룰) · 중:내 제스처 분류 학습 · 심:제스처 가상버튼/제어

**4 ML 개념 실험실** — 초:지도학습 복습 · 중:**비지도 군집화(K-means)** · 심:임베딩 2D(PCA)·평가지표
> 보완: 학생 자기 사진 자동 군집화로 "라벨 없이 묶임" 체감.

**5 생활 인식** — 초:QR 찾기 · 중:글자 인식(OCR) · 심:명함/문서 정보 추출 미니앱

## 7. 대상별 학습 경로
- **저학년(초등 저~중)** — 보고/만지고/놀며: 3초급(가위바위보) → 0초급(비추면 인식) → 5초급(QR) → 0중급(내 AI 첫 제작)
- **중·고등** — 만들고 값 바꾸기: 0중급 → 3중급 → 2초중급 → 5중급 → 1
- **대학·성인** — 개념·평가·연동: 4 → 2심화(MLP·센서) → 1심화 → 0심화

## 8. SW 아키텍처 (공통 코어)
재사용할 코어 컴포넌트(현재 앱에서 분리 예정):
- **Shell/Sidebar** — config(NAV_GROUPS) 기반 자동 생성, 영역·뱃지·활성표시
- **FeatureInfo + Theory** — 학습/알고리즘 정보 카드 (config: type/model/desc/THEORY)
- **StepNav** — 단계 진행 표시·상태 연동
- **Trainer 코어** — `trainHead`(전이학습), 학습곡선, 혼동행렬, 도넛 게이지
- **Media 코어** — 카메라/마이크 라이프사이클(`stopAllActivity`), 보안컨텍스트 안내
- **Viz 코어** — 게이지/도넛/막대, 파라미터 슬라이더
- **Test 하니스** — jsdom + 모델 스텁(회귀 테스트)
→ 모듈은 위를 import 하고 **기능 config + 모델 로직**만 추가.

## 9. 개발 로드맵
- **0단계** 공통 코어 분리(현재 앱 → 재사용 가능한 템플릿/프레임워크화)
- **1차 출시** 모듈 3(제스처) — 수요·상호작용·코드 재사용성 최상
- **2차** 모듈 2(데이터·AIoT) — 에듀이노 정체성/차별화
- **3차** 모듈 5(생활) → **4차** 모듈 4(개념) → **5차** 모듈 1(텍스트, 보완)

## 10. 1차 MVP 후보 상세 — 모듈 3 (제스처·동작)
- **초급**: 손 21관절 실시간 오버레이 + 가위바위보 인식(룰 기반) + LED/도넛 피드백
- **중급**: 내 제스처 3~4종 수집 → 전이학습 → 실시간 분류(도넛/막대)
- **심화**: 제스처로 가상 버튼/게임 제어, 동작 시퀀스
- **상호작용**: 몸으로 참여, 가위바위보 대결, "멈추기/따라하기" 게임
- **기술**: `@tensorflow-models/hand-pose-detection` + 공통 trainHead 재사용
- **MVP 범위**: 초급(체험) + 중급(학습) 우선, 심화는 2차

## 11. 향후 — 하드웨어 키트 연계 (메모, 추후)
- 웹 AI ↔ Web Serial/BLE ↔ 보드+Grove(센서/서보/LED)로 "실물 제어·행동예측" 확장.
- 소형 이동 베이스 + 경량 로봇팔("데스크탑 더미봇") 데모는 SW 안정화 후 진행.

## 12. 결정 필요
1. 1차 모듈 확정(추천: **모듈 3 제스처**)
2. 진행 방식: (A) 공통 코어 분리 먼저 vs (B) 첫 모듈 바로 제작
3. 모듈을 **현재 저장소에 탭으로 추가** vs **신규 저장소/배포로 분리**
